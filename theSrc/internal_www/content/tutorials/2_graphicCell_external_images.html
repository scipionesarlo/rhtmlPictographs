Single Graphic Cell External Image Usage Tutorial

External images can be used in graphic cells.

Given the aspect ratio of the external image may not fit the assigned image slot, the widget must make some scaling decisions.

Consider this external image of a 600x900 green rectange:

Consider placing that external image inside several different graphic cell dimensions:

It fits: 
url:/images/green_rectangle_600_900.jpg
It does not fit (image slot too wide): 
url:/images/green_rectangle_600_900.jpg
It does not fit (image slot too tall): 
url:/images/green_rectangle_600_900.jpg

There are two ways to address this if you want the external image to completely fill the provided image slot:

  • Option 1: use the "slice" feature of SVG aspect ratios: This tells the SVG engine to fill the slot with the image and try to get the aspect ratio as close as possible, and its ok to crop the image a bit.
  • Option 2: use the "none" feature of SVG aspect ratios: This tells the SVG engine to fill the slot with the image by distorting it, completely ignoring the aspect ratio of the image.

You can also "shift" the image up down left and right so all extra space will appear on one side of the image:

  • Option 3: use the "min/mid/max" setting of SVG aspect ratios: This tells the SVG engine to move the image in specific directions so all extra space occurs one or more sides of the image.

Option 1: use the "slice" feature of SVG aspect ratios

Valid values are found here : https://developer.mozilla.org/en/docs/Web/SVG/Attribute/preserveAspectRatio Basically we are changing the "preserveAspectRatio" value of the <svg:image> tag: It does not fit (image slot too wide): 
{
      "variableImage": {
        "type": "url",
        "url": "/images/green_rectangle_600_900.jpg",
        "preserveAspectRatio" : "xMidYMid slice"
      }
    }
It does not fit (image slot too tall): 
{
      "variableImage": {
        "type": "url",
        "url": "/images/green_rectangle_600_900.jpg",
        "preserveAspectRatio" : "xMidYMid slice"
      }
    }

Option 3: use the "min vs mid vs max" settings of SVG aspect ratios

Valid values are found here : https://developer.mozilla.org/en/docs/Web/SVG/Attribute/preserveAspectRatio Basically we are changing the "preserveAspectRatio" value of the <svg:image> tag: It does not fit (image slot too wide): So shift it to the left 
{
      "variableImage": {
        "type": "url",
        "url": "/images/green_rectangle_600_900.jpg",
        "preserveAspectRatio" : "xMinYMid meet"
      }
    }
It does not fit (image slot too wide): So shift it to the right 
{
      "variableImage": {
        "type": "url",
        "url": "/images/green_rectangle_600_900.jpg",
        "preserveAspectRatio" : "xMaxYMid meet"
      }
    }
It does not fit (image slot too tall): So shift it up 
{
      "variableImage": {
        "type": "url",
        "url": "/images/green_rectangle_600_900.jpg",
        "preserveAspectRatio" : "xMidYMin meet"
      }
    }
It does not fit (image slot too tall): So shift it down 
{
      "variableImage": {
        "type": "url",
        "url": "/images/green_rectangle_600_900.jpg",
        "preserveAspectRatio" : "xMidYMax meet"
      }
    }
